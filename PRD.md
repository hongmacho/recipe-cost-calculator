# 레시피 원가 계산기 - PRD (Product Requirements Document)

## 1. 개요

**프로덕트명**: 레시피 원가 계산기  
**슬러그**: recipe-cost-calculator  
**핵심 가치 제안**: 카페·음식점·도시락 사업자가 엑셀 없이 레시피별 원가를 자동으로 계산하고, 마진을 시뮬레이션하며, 수익율을 추적하는 웹 앱  
**타겟 출시**: MVP (7개 화면, 기본 기능)  

---

## 2. 타겟 페르소나

### 주요 페르소나: 소규모 음식 사업자
- **직업**: 카페·음식점 사장, 도시락 사업자, 요리 프리랜서
- **나이**: 25~55세
- **기술 수준**: 중급 (엑셀 능숙, 웹앱 사용 가능)
- **월 통증**: 
  - 레시피별 원가를 엑셀로 매번 수동 계산 (시간 낭비 33시간/월)
  - 재료 가격 변동 시 수동으로 전체 레시피 재계산
  - 판매가 vs 원가 비교 번거로움
  - 수익율 추이 파악 불가능
- **기대**: "원가 계산만 간단하게. 복잡한 POS는 원하지 않음"
- **사용 빈도**: 주 2~3회 (신규 레시피 추가 또는 가격 변동 반영)

## 3. MoSCoW 기능 목록

### MUST-HAVE (8개)

#### 1. 재료 데이터베이스 관리
- **설명**: 카테고리별 재료 등록 및 관리
- **서브기능**:
  - 재료명, 단가(원/단위), 단위(kg, g, 개, 병 등) 입력
  - 재료 수정/삭제 (연관된 레시피에 실시간 반영)
  - 카테고리 분류 (곡류, 육류, 유제품, 향신료 등)
- **UI 텍스트**: "재료 데이터베이스" / "재료 추가" / "단가 변경" / "단위 선택"

#### 2. 레시피 정의 (킬러 기능)
- **설명**: 레시피명, 재료 선택 & 수량 입력
- **서브기능**:
  - 재료 검색/선택
  - 수량 입력 (e.g., 밀가루 300g)
  - 수량 자동 단위 변환 (200g + 0.5kg = 700g)
  - 레시피 템플릿 저장 (복제 기능)
- **UI 텍스트**: "새 레시피" / "재료 추가" / "수량" / "단위 선택" / "저장"

#### 3. 원가 자동 계산 (킬러 기능)
- **설명**: 선택된 재료 & 수량 → 총 원가 자동 계산
- **서브기능**:
  - 실시간 계산 (사용자가 수량 입력하면 즉시 원가 업데이트)
  - 재료별 원가 세부 표시 (e.g., 밀가루: 2,000원)
  - 단위 변환 오류 감지 및 경고
  - 원가 단가 자동 도출 (총 원가 / 수량 = 1개당 원가)
- **UI 텍스트**: "총 원가" / "재료별 원가" / "원가 단가"

#### 4. 판매가 & 수익율 시뮬레이션 (킬러 기능)
- **설명**: 판매가 입력 → 수익율/마진 자동 계산
- **서브기능**:
  - 판매가 입력 필드
  - 수익액 표시 (판매가 - 원가)
  - 수익율 표시 (수익액 / 판매가 × 100%)
  - 목표 수익율 설정 → 목표에 맞는 판매가 역산
  - 마진율 표시 (수익액 / 원가 × 100%)
- **UI 텍스트**: "판매가" / "수익액" / "수익율" / "마진율" / "목표 수익율" / "추천 판매가"

#### 5. 레시피 목록 & 검색 (MUST-HAVE: 검색/필터)
- **설명**: 등록된 레시피 목록 조회, 검색/필터
- **서브기능**:
  - 레시피 검색 (레시피명, 재료명)
  - 카테고리 필터 (음료, 베이커리, 도시락 등)
  - 정렬 (최근 추가순, 원가 낮은순, 높은순)
  - 목록에서 원가/판매가/수익율 미리보기
  - 페이지네이션 (20개/페이지)
- **UI 텍스트**: "검색" / "필터" / "카테고리" / "정렬" / "레시피 목록"

#### 6. 통계 & 대시보드 (MUST-HAVE: 통계/대시보드)
- **설명**: 경영 지표를 한눈에 파악
- **서브기능**:
  - 평균 원가, 평균 판매가, 평균 수익율 (전체 레시피 기준)
  - 재료 사용 빈도 (TOP 5)
  - 원가별 레시피 분포 (차트)
  - 수익율 분포 (차트: 60% 이상, 40~60%, 40% 이하)
  - 월간 레시피 추가 추이
- **UI 텍스트**: "대시보드" / "통계" / "평균 원가" / "평균 판매가" / "인기 재료" / "수익율 분포"

#### 7. 재료비 변동 추적 (히스토리)
- **설명**: 재료 가격 변경 시 자동 기록, 추이 분석
- **서브기능**:
  - 재료 가격 변경 이력 (날짜, 이전 단가, 현재 단가)
  - "지난달 vs 이번달" 비교 (영향받는 레시피 목록)
  - 가격 변동이 수익율에 미치는 영향 분석
  - CSV 내보내기 (가격 변동 히스토리)
- **UI 텍스트**: "가격 변동 기록" / "지난달 원가" / "이번달 원가" / "변동율" / "영향받는 레시피"

#### 8. 빈 상태 & 에러 처리 (MUST-HAVE: 빈 상태)
- **설명**: 사용자 친화적 안내 메시지
- **서브기능**:
  - 재료 없음: "재료를 먼저 추가하세요" (+ 추가 버튼)
  - 레시피 없음: "첫 레시피를 만들어보세요" (+ 튜토리얼 링크)
  - 검색 결과 없음: "해당 검색어에 맞는 레시피가 없습니다"
  - 단위 오류: "유효하지 않은 단위입니다" (+ 제안)
  - 로딩 상태: 스켈레톤 UI 또는 로딩 스피너
- **UI 텍스트**: "재료가 없습니다" / "레시피를 만들어보세요" / "검색 결과 없음" / "오류가 발생했습니다"

### SHOULD-HAVE (3개)

#### 9. 레시피 복제 & 배리에이션
- **설명**: 기존 레시피 기반으로 신규 레시피 빠르게 생성
- **서브기능**:
  - "복제" 버튼 (기존 재료·수량 복사)
  - 배리에이션 관리 (원본 레시피 vs 변형 버전)
  - 배리에이션끼리 비교 (원가 차이 표시)

#### 10. 대량 재료 수입 (CSV 업로드)
- **설명**: 엑셀 파일로 재료 일괄 등록
- **서브기능**:
  - CSV 템플릿 다운로드
  - 파일 검증 (필수 컬럼 확인)
  - 중복 처리 (덮어쓰기 vs 병합)

#### 11. 재료별 공급처 메모
- **설명**: 재료마다 공급처/전화/최저가 기록
- **서브기능**:
  - 공급처명, 전화, 최저가 메모
  - 공급처별 가격 비교

---

### COULD-HAVE (3개)

#### 12. 팀 공유 & 권한 관리
- **설명**: 여러 사용자가 같은 가게의 데이터 공유
- **서브기능**:
  - 사용자 초대 (이메일)
  - 역할 (관리자, 편집자, 조회자)

#### 13. 클라우드 백업
- **설명**: 데이터 클라우드 동기화 (로컬은 기본, 클라우드는 선택)
- **서브기능**:
  - Google Drive 연동 (선택사항)
  - 자동 백업

#### 14. AI 기반 재료 제안
- **설명**: 레시피 입력 시 유사 재료 추천
- **서브기능**:
  - "최근 사용 재료" 빠른 추가
  - 재료명 자동완성

## 4. 데이터 모델 (초안)

### Ingredient (재료)
```
id: UUID
name: string (e.g., "밀가루")
category: enum (곡류, 육류, 유제품, 향신료, 등)
unitPrice: number (원, 예: 2000)
unit: enum (kg, g, 개, 병, 팩, 등)
supplier?: string (공급처)
supplierPhone?: string
updatedAt: timestamp
```

### Recipe (레시피)
```
id: UUID
name: string (e.g., "초콜릿 머핀")
category?: string (음료, 베이커리, 도시락, 등)
totalCostWon: number (계산값)
soldPrice?: number (판매가, 선택)
margin?: number (수익율, 계산값)
markup?: number (마진율, 계산값)
createdAt: timestamp
updatedAt: timestamp
```

### RecipeIngredient (레시피-재료 연결)
```
id: UUID
recipeId: UUID (FK)
ingredientId: UUID (FK)
quantity: number (e.g., 300)
unit: enum (최종 단위)
costForThisRecipe: number (계산값, e.g., 6000원)
```

### IngredientPriceHistory (재료비 변동)
```
id: UUID
ingredientId: UUID (FK)
oldPrice: number
newPrice: number
changedAt: timestamp
```

## 8. Flaw Mitigations (아이디어 기반 위험 요소 해결)

### 1. 데이터 입력 정확도 (사용자 수동 입력)
- **위험**: 사용자 입력 오류 → 원가 계산 부정확
- **대책**:
  - 입력값 검증 (음수 방지, 단위 오류 감지)
  - 재료 검색 시 정확한 이름 추천
  - 가격 히스토리 자동 추적으로 "지난달 vs 이번달" 비교 기능
  - 계산 전에 사용자가 입력값 확인할 수 있는 UI

### 2. 공급처 연동 부재 (바코드/API 연동 없음)
- **위험**: 수동 입력만 → 확장성 제한
- **대책**:
  - MVP는 수동 입력으로 시작 (단순성 우선)
  - 공급처 메모 기능으로 향후 연동 기반 마련
  - CSV 대량 수입 기능으로 엑셀→앱 마이그레이션 용이

### 3. Toast 등 경쟁자 대비 약세
- **위험**: 시장 점유 어려움
- **대책**:
  - Toast: 전사업 POS (복잡, 비쌈, 70% 고가)
  - 이 앱: 원가 계산만 집중 → 사용 난이도 70% 낮음
  - 초기 타겟: "원가 계산만 원하는 소규모 사업자"
  - UI 속도와 단순성으로 차별화
