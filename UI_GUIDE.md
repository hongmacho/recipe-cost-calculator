# 레시피 원가 계산기 - UI/UX 가이드

## 1. 디자인 철학

- **단순함**: 사용자가 5초 내에 필요한 기능을 찾을 수 있어야 함
- **명확함**: 모든 숫자는 단위와 함께 표시 (원가: 5,000원)
- **응답성**: 모바일 600px 이상 완벽하게 작동
- **한국어 우선**: 모든 UI 텍스트는 한국어 (영어 절대 금지)
- **시각적 계층**: 중요한 정보가 눈에 띄도록 배치

## 13. 반응형 레이아웃

### 중단점 (Breakpoints)
```
xs: 320px (기본, 모바일)
sm: 640px (작은 태블릿)
md: 768px (태블릿)
lg: 1024px (데스크톱)
xl: 1280px (큰 데스크톱)
```

### 레이아웃 예시

#### 모바일 (< 640px)
```
- 풀 너비 한 컬럼 레이아웃
- 사이드바 네비게이션 (햄버거 메뉴)
- 버튼: 풀 너비
- 카드: 풀 너비 스택
- 테이블: 스크롤 가능 또는 카드 뷰로 변환
```

#### 테블릿 (640px ~ 1024px)
```
- 2열 그리드
- 상단 네비게이션
- 카드 그리드 2x2
```

#### 데스크톱 (>= 1024px)
```
- 3~4열 그리드
- 상단 네비게이션
- 사이드바 (고정, 선택사항)
```

## 15. 접근성 (WCAG 2.2 AA)

### 색상 대비
```
텍스트: 최소 4.5:1
큰 텍스트 (18px+): 최소 3:1
UI 컴포넌트: 최소 3:1
```

### 포커스 표시
```
포커스 아웃라인: 2px solid #3b82f6(파란)
모든 대화형 요소: Tab 키로 이동 가능
포커스 순서: 논리적 순서 유지
```

### 키보드 네비게이션
```
Tab: 다음 요소
Shift+Tab: 이전 요소
Enter: 액션 실행
Escape: 모달 닫기
화살표 키: 메뉴 항목 네비게이션
```

### 스크린 리더
```
alt 텍스트: 모든 이미지에 설명
aria-label: 아이콘 버튼에 레이블
role="alert": 긴급 메시지
aria-live="polite": 동적 콘텐츠 업데이트
```

거리: 16px ~ 32px

불투명도: 1 → 0.5 → 1

### 호버 상태
```
지속: 0.15s
타이밍: ease-in
예시: 버튼, 링크 색상 변경
```
